Databricks Platform Features Used

FeatureHow It's UsedWhat It Delivers
Unity Catalog PII column tags · Auto lineage · Foreign catalogs · Sensitivity tiers · Per-persona grants + column masks (17 cols, DPDP §5(2)) + bronze-is-pipeline-only fence + row filters (§16 residency + CMO persona purpose-scope) enforcing CCO/GC/CMO/CFO boundaries at query time One governance boundary across the estate — an SA or regulator gets an answerable question in a single query instead of navigating a patchwork of app-level checks. Enforcement is at query time, so no UI path can leak what UC has blocked.
Regulation packs (Phase 0 framework) All DPDP-specific values (9 rules, 3 notice bodies, per-purpose retention, allowed-country residency list, 5 region PII patterns, DSR SLAs) live in regulations/dpdp_2023/. governance_core/ hosts the regulation-agnostic code. One env var (REGULATION_PACK) selects the active pack; zero code changes needed to add UK GDPR / CCPA / PIPEDA / LGPD / APPI. Same lakehouse + same personas + same dashboards, different regulation. Makes the accelerator a Unity-Catalog implementation framework, not a single-jurisdiction compliance product. See modular_framework.html for the 4-layer architecture and effort-to-add-a-regulation matrix.
system.access.audit Every SQL query + dashboard view + Genie question captured with user + timestamp (DPDP §18 evidence). CCO Genie space surfaces it via sample questions DPDP §18 audit evidence available on demand. "Who accessed patient X's record last week?" answered live in seconds — the difference between surviving a regulator inquiry and scrambling to reconstruct logs from app traces.
Lakeview Dashboards + Genie (persona-sliced) 4 persona-specific dashboards sliced from the master (idempotent PATCH on redeploy) + 4 Genie spaces scoped to that persona's data_sources. Dashboards published with embed_credentials=false so queries run as the viewer and UC enforces Every executive (CCO/GC/CMO/CFO) gets a personal view tailored to their questions — not one firehose dashboard. Natural-language agents collapse "file a ticket, wait two days, get a spreadsheet" into a five-second Genie answer.
Lakeflow Declarative Pipelines Bronze → Silver medallion with 16 DQ expectations · PII classification pipeline · metrics to event log Data quality becomes an always-on control, not a periodic review. The 16 expectations auto-emit pass/fail metrics to the event log, so a DPO can prove "our controls ran every night and caught these rows" without custom observability code.
Agent Bricks + AI Functions DPIA auto-generation · Compliance Q&A · ai_classify/ai_extract for unstructured PII DPIA production drops from ~3 months of legal + IT + compliance coordination to ~30 seconds of agent output + ~1-2 hours of human review. For an enterprise running 20+ DPIAs a year, that's ₹1.5–3 crore of external-consulting spend that goes away.
Lakehouse Federation Foreign catalogs for Postgres, Snowflake, BigQuery — query in place, no ETL Data residency stays compliant without engineering work — the source system never has to move its data. For cross-border concerns under DPDP §16, this is often the answer: query where the data already lives.
Lakeflow Connect Managed ingestion for Salesforce, Workday, ServiceNow, SharePoint Onboarding a new SaaS source goes from a 2-sprint engineering project to a configuration step. Schema drift, OAuth renewals, pagination and rate-limit handling are platform responsibilities — ingestion stops being a product your team maintains.
Auto Loader Incremental file ingestion from S3/ADLS/GCS with schema evolution Exactly-once file ingestion without your team tracking what's been processed. Schema evolution is automatic — partner uploads a new column, your pipeline keeps running instead of paging an engineer at 2am.
AI/BI Dashboard + Genie 10-page compliance dashboard + NL query support · 4 persona-scoped Genie spaces Compliance insight accessible to business users without SQL literacy. A CMO asks "can I email this customer?" in English/Hindi and gets a yes/no answer — no analyst ticket, no wait, no raw-PII exposure.
Delta Lake (CDF) Immutable consent log · Withdrawal propagation · Time travel for audit Real-time consent withdrawal propagation — DPDP §8(7) "without undue delay" is satisfied by the platform, not by a nightly batch job. Downstream marketing / ML systems auto-invalidate the moment a principal withdraws; no polling, no stale cache.
Delta Sharing Read-only audit share (dpdp_audit_view_share) — 4 objects: personal_data_register, compliance_gaps, notice_versions, consent_coverage_summary External auditors query compliance evidence from their own workspace — no local accounts to provision, no VPN, no orphan credentials to clean up after the audit. Revoke in one command when the engagement ends.
Databricks Asset Bundle Full platform as IaC — one command deploy, one command destroy Reproducibility becomes free. A teammate stands up the entire compliance stack on a fresh workspace in minutes instead of days; every environment looks identical; drift between dev and prod stops existing as a category.
Persona Orchestrator (scripts/setup_all_personas.py) One command stands up the full persona layer: slice → Genie → attach banners → SCIM users → UC grants → warehouse/dashboard/Genie ACLs. Auto-detects workspace/warehouse/deployer. Idempotent end-to-end Governance is deployed the same way code is — scriptable, reviewable, reproducible. Onboarding a new persona (e.g. a Chief Risk Officer) becomes a code change with a PR, not a week of click-ops that quietly drifts between environments.
GitHub Actions + persona_boundary_test On every push/PR: bundle validate + Python AST parse + HTML tag-balance + live 4/4 persona-boundary check against the workspace Governance is continuously proven, not annually audited. If a future change breaks a persona boundary, CI fails the PR before it merges — regulator-grade enforcement with a commit-level feedback loop.
Lakebase (Production) Consent OLTP tier with sub-second writes and read-after-write consistency Consent capture latency goes from "acceptable" to "imperceptible" — a principal withdrawing on your mobile app sees the new state reflected across downstream systems in under a second. One Databricks surface handles both the OLTP write and the lakehouse analytics.

Note on numbers. Counts shown in the module cards and metric tiles (33 PII findings, 14 auto-classified, 135 gaps, 72 critical, 52 high, 1,000 consent events, 292 principals, 286 marketing eligible, 9 rules) are illustrative — they come from the synthetic POC dataset on the dpdp_poc workspace and are reproducible via the SQL in docs/how_to_test.html. In a real deployment the absolute counts will differ with the customer's data estate and PII density; the transferable signal is the shape of the ratios (critical-to-total gap mix, auto-classify rate, consent-grant rate), not the specific numbers. The ₹1.5–3 cr DPIA-consulting savings in the platform-features table is an industry-range estimate drawn from Big-4 readiness benchmarks; actual per-enterprise savings vary with scale, SDF status, and current tooling.
